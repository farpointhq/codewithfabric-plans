Team Invite Flow
Phase 1: Send Invite
POST /api/team/invite
1 Auth + getTeamContext() → verify OWNER or ADMIN
2 Validate email, normalize to lowercase
3 Anti-hijacking check: invitee not in ancestor team
4 Check for duplicate pending invite to same email
5 Create TeamInvitation (7-day expiry, secure token)
6 Send invite email via Resend
Phase 2: Accept Invite
POST /api/team/invitations/[token]/accept
1 Auth required — user must be signed in
2 Validate token: PENDING, not expired, email matches session user
3 Determine invitee state (5 scenarios)
4 Execute migration in prisma.$transaction
5 Generate API key via LiteLLM if user has none
6 Mark invitation ACCEPTED, notify team owner
Supporting Routes
GET /api/team/invitations
Auto-expire past-due invitations, return list with email, role, status, dates
POST /api/team/invitations/[token]/revoke
OWNER or ADMIN only. Validate token belongs to team. Set status → REVOKED
