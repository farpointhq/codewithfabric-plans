Summarization Algorithm Flow
When a new block is finalized, walk backwards to find blocks that now have enough lookahead context to be summarized.
1
Trigger: New Block Finalized
— fires after each tool result, raw output, or thinking block completes
When the loop-orchestrator finalizes a block (tool result arrives, thinking completes, raw output finishes), it calls EagerSummarizationService.onBlockFinalized(blockIndex).

This is the only trigger — no timers, no polling, no message-level hooks.
↓
2
Walk Back: Find Summarizable Blocks
— check B[n-2] and earlier that now have 2+ next blocks
Starting from blockIndex - 2, walk backwards through the flat block sequence:
For each block B[i], check:
SUMMARIZE if:
• Block type is thinking, tool_result, or raw_output
• Block has ≥ MIN_BLOCK_TOKENS (100)
• Block not already summarized (status ≠ "complete")
• Block has ≥ 2 next blocks available OR is end of response
SKIP if:
• Block is a user message (never summarized)
• Block is a tool_call (too small, keep verbatim)
• Block is already summarized
• Block is < 100 tokens (not worth the LLM call)
↓
3
Build Context Window
— assemble the prompt for each block to summarize
For each block B[i] to summarize:

a) First user message — always include (unless it's in the 5-block window)
b) Previous 5 blocks B[i-5]…B[i-1]:
  • Use summary text if block already summarized
  • User messages and tool calls: verbatim
  • Unsummarized tool results: truncate to 500 tokens
c) Target block B[i]: full raw content
d) Next 2 blocks B[i+1]…B[i+2]: truncate each to 500 tokens
↓
4
Summarize via fabric-small
— single-pass LLM call with /no_think
Single-pass for all block types — no Chain of Density (CoD hangs on fabric-small).

• System prompt: role + target word count
• User prompt: assembled context + target + instruction
• Append /no_think to disable Qwen thinking
• Set reasoning_effort: "off"
• 45s timeout per block

All eligible blocks in a batch fire in parallel (up to 15 concurrent).
↓
5
Persist Immediately
— each block saved as soon as its summary arrives
Each block's summary is persisted to messageData.blockSummaries[blockId] immediately — no batch waiting.

The prompt-builder swaps these summaries for raw content when context utilization reaches 60%.
Timing: When Each Block Gets Summarized
Blocks are summarized as soon as they have 2 next blocks of lookahead context. This means summarization happens ~2 blocks behind the conversation head.

summarized when B[5] arrives

B[6] thinking (250 tok)
