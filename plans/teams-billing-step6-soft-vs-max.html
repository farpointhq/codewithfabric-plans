Why we don't need an app-level fail-open wrapper: the proxy already distinguishes soft alerts from hard blocks.
Key insight driving the plan
LiteLLM has three independent budget fields on each key. We treat them as three switches for the rollout: soft_budget for stage 2 (observability), max_budget for stage 3 (enforcement), and budget_duration decoupled from both (reset cadence, can be set any time). No additional proxy wiring required.
