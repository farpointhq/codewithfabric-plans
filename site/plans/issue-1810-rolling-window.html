Rolling Window: 5 Previous + 2 Next
To summarize block B[n], we build a context window: the 5 blocks before it, the target block itself, and the 2 blocks after it. User messages and tool calls are context-only (never summarized). Already-summarized blocks use their compressed form.
Example: Summarizing B[9] (thinking block)
B[0]
👤 Read config.json and requirements.txt. Explain them briefly. FIRST USER MSG
⋮ B[1]–B[3] omitted (outside window)
PREVIOUS 5 BLOCKS (context)
B[4]
🔧 Read(requirements.txt) 15 tok PREV-5
B[5]
📦 Python deps: Flask 2.3, requests, numpy, pandas + 12 others for ML pipeline 28→18 tok PREV-4 (summarized)
B[6]
📦 Identified config.json as npm config and requirements.txt as Python ML deps 250→22 tok PREV-3 (summarized)
B[7]
📦 Explained: config.json = Node.js app v2.1.0 w/ React; requirements.txt = Flask ML backend 220→30 tok PREV-2 (summarized)
B[8]
👤 Now refactor the Flask app to use blueprints. 12 tok PREV-1
TARGET BLOCK (to summarize)
B[9]
💭 The user wants to refactor app.py to use Flask blueprints. I need to read the current app structure first, understand the route layout, then create separate blueprint modules for API routes, page routes, and error handlers... 300 tok TARGET
NEXT 2 BLOCKS (context)
B[10]
🔧 Read(app.py) 12 tok NEXT-1
B[11]
📄 from flask import Flask, render_template... (1,800 tokens of raw content) 1800 tok NEXT-2
Key insight: B[5], B[6], B[7] were already summarized from an earlier pass. We use their compressed summaries as context (28+22+30 = 80 tokens), not the raw content (520+250+220 = 990 tokens). This is a 12x context compression for those blocks.
Rules:
• User messages (B[0], B[8]) are included verbatim as context — never summarized
• Tool calls (B[4], B[10]) are included verbatim — they're small and important for understanding what was done
• Already-summarized blocks use their summary text, not raw content
• First user message (B[0]) is always included, even if outside the 5-block window
• Bias: 5 previous (heavier history) + 2 next (lighter lookahead)
Window Size Comparison
1 PREV + 0 NEXT (current)
Only sees the immediately preceding block.
Misses: conversation goal, tool results, reasoning chain.
Summaries are context-poor and lose the thread.
5 PREV + 2 NEXT (proposed)
Sees the full chain of reasoning.
Previous summaries compress older context (telescoping).
Lookahead adds what the block leads to.
